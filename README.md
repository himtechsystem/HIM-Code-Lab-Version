## 📖 The Vision

**HIM Code** is the first AI-Native development environment that treats AI Agents not as chatty assistants, but as **Autonomous Execution Units**. By leveraging a **Fractal Organization** structure and a **Semantic Program Runtime**, HIM Code transforms user intent into verifiable, atomic actions.

---

## 💎 Core Architecture (Capability Map)

### 1. Native Workbench Integration
Unlike traditional plugins, HIM Code is a deep fork of VSCodium. We've re-engineered the **VS Code Workbench** to host a high-performance AI core with a "Cyberpunk-Anime" aesthetic.

<details>
<summary><b>📂 View Prompt for Cloning</b></summary>
<br />

```text
"Re-engineer the VS Code Auxiliary Bar to host a native React-based AI panel. 
Implement a multi-session manager using 'retainContextWhenHidden' and CSS Z-index 
stacking to ensure that background AI streams never disconnect during session switching."
```

</details>




### 2. Fractal Orchestration Engine (The Brain)
HIM Code introduces Fractal Organization. An Orchestrator doesn't just assign roles; it recursively spawns sub-agents as task complexity increases, ensuring each agent remains focused on a specific "World Scope."
<details>
<summary><b>📂 View Prompt for Cloning</b></summary>
<br />
code
Text
"Implement a Fractal Agent Registry. When a task exceeds the 'Complexity Threshold' 
(e.g., >20 files), the parent agent must spawn sub-specialists. 
Define a recursive routing logic where a detail change (like CSS color) is 
automatically dispatched to the deepest relevant leaf-node agent."
</details>


### 3. HIM-Assembly: Semantic Program Runtime
We treat the LLM as a Compiler/Interpreter. Instead of fuzzy chat history, HIM Code executes a structured .him script—a Natural Language Instruction Set with unique OpCodes and control flows.
<details>
<summary><b>📂 View Prompt for Cloning</b></summary>
<br />
code
Text
"Create a Semantic Program Runtime. The AI must output its plan in a JSON-based 
instruction set (OpCodes). Implement a 'Fetch-Decode-Execute' loop where the 
IDE validates each step against 'Atomic Rules' (<100 lines per CRUD) before 
handing code to the execution engine."
</details>

	
### 4. Code as Action: Unified Python Engine
HIM Code interacts with the physical file system and terminal solely through a persistent Python REPL. This ensures a unified interface for all models (MiniMax, Claude, or local Gemma).
<details>
<summary><b>📂 View Prompt for Cloning</b></summary>
<br />
code
Text
"Build a persistent Python Sidecar service using child_process.spawn. 
Inject a HIM-SDK into the Python environment to map 'print()' calls to 
UI chat messages and 'os' operations to authenticated FileService calls. 
All AI actions must be captured via Stdout redirection."
</details>

### 5. Local-First: Zero Token Cost
Optimized for the Mac M1/M2/M3 architecture, HIM Code supports embedding Gemma 2 2B/9B via an internal llama-server, providing 100% privacy and zero operational costs.
